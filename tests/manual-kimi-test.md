# 手动测试 Kimi AI 分析功能

## 测试步骤

### 1. 启动开发服务器

```bash
npm run dev
```

访问 http://localhost:3000

### 2. 创建新的评估

1. 点击 "开始新评估"
2. 填写项目信息（名称、行业、融资阶段）
3. 完成所有评估维度（市场、产品、团队、财务、执行）
4. 提交评估

### 3. 进入评估结果页

完成评估后，会自动跳转到结果页（或从列表点击评估）

### 4. 点击 "AI深度分析"

在评估结果页中，点击 "AI深度分析" 按钮

## 预期结果

### ✅ 成功情况

每个模块应该：
1. 显示 "AI正在分析中" 的加载状态（约 2-5 秒）
2. 显示分析结果（中文，专业、具体的建议）
3. 结果包含项目相关的具体信息

1. 显示错误消息（红色提示框）
2. 错误消息可能是：
   - "AI分析失败，请重试" - 网络问题或 API 错误
   - "本月AI分析额度已用完" - 配额问题
   - "AI分析功能需要Pro或Premium订阅" - 权限问题

## 错误排查

### 如果 AI 分析失败：

1. **检查浏览器控制台 (F12 → Console)**
   - 查找红色错误消息
   - 查看网络请求 (Network tab)

2. **检查网络请求**:
   - 请求 URL: `https://jyjnadjvhoeyudltvpfy.functions.supabase.co/kimi-analysis`
   - 请求方法: POST
   - 状态码应该是: 200
   - 响应应该有: `{ analysis: "..." }`

3. **常见错误**:
   - `401`: API Key 无效
   - `404`: Function 未部署
   - `500`: 服务器错误或 API Key 未配置

### 如果结果显示异常：

1. **检查 prompt** (在 AIAnalysis.jsx 中):
   - Prompt 是否包含正确的项目信息？
   - Prompt 是否用英文编写？（应为英文，因为 Kimi 对英文 prompt 理解更好）

2. **检查 API 响应**:
   - 响应是否包含完整的分析？
   - 是否出现截断或乱码？
